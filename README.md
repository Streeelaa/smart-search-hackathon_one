# 🔍 Персонализированный умный поиск продукции — zakupki.mos.ru

Персонализированная поисковая система для Портала поставщиков Москвы, работающая на **реальных данных госзакупок**.

> **542 993 товара (СТЕ)** · **2 010 224 контракта** · **4 192 организации** · **6 506 категорий**

## Ключевые возможности

| Возможность | Реализация | Пример |
|---|---|---|
| **BM25 полнотекстовый поиск** | SQLite FTS5 (unicode61) с взвешиванием полей | title:10, category:5, attributes:1 |
| **Семантический поиск** | sentence-transformers (paraphrase-multilingual-MiniLM-L12-v2), 6506 эмбеддингов категорий | «канцтовары» → Клеи, Карандаши, Наборы |
| **Автокоррекция опечаток** | rapidfuzz + pymorphy3 валидация + ручные корректировки | перчтки → перчатки, мантор → монитор |
| **Морфология** | pymorphy3 — лемматизация для русского языка | ноутбуки = ноутбук |
| **Синонимы** | 158 групп (аббревиатуры, жаргон, проф. термины) | МФУ → многофункциональное устройство |
| **Персонализация** | Мультипликативный буст по контрактной истории | Аптека: «шприц» → иммунодепрессанты (+146%) |
| **3-уровневое кэширование** | Токены → LRU(512) → Результаты(256) | Повторные запросы < 0.1мс |
| **Фасетная фильтрация** | Категории товаров из результатов поиска | 6506 категорий с фильтром |
| **Стоп-слова** | Фильтрация  предлогов/союзов из FTS-запросов | «для», «на», «по» и др. |
| **Подсветка совпадений** | `<mark>` теги на совпавших токенах | **ноутбук** Lenovo ThinkPad |

## Архитектура (Pipeline)

```
Запрос пользователя
  │
  ├── [1] Токенизация + морфологическая лемматизация (pymorphy3)
  │
  ├── [2] Автокоррекция опечаток:
  │       - rapidfuzz fuzzy matching (score_cutoff=70, ±3 символа)
  │       - pymorphy3 валидация (не исправлять реальные слова)
  │       - Ручные корректировки (мантор→монитор, компютер→компьютер)
  │
  ├── [3] Расширение синонимами (158 групп из data/synonyms.json)
  │       + raw-токены синонимов для сохранения множественных форм
  │
  ├── [4] Фильтрация стоп-слов (предлоги, союзы)
  │
  ├── [5] FTS5 BM25 retrieval (title ×10, category ×5, attrs ×1)
  │
  ├── [6] Семантическое расширение (условное):
  │       - Активируется когда FTS5 вернул < limit×3 результатов
  │       - sentence-transformers кодирует запрос → cosine similarity
  │       - Находит похожие категории → добавляет товары из них
  │
  ├── [7] Scoring:
  │       - BM25 score (fulltext relevance)
  │       - Title match bonus (+5.0 за точное совпадение фразы)
  │       - Category match bonus (+1.0)
  │       - Персонализация: score × (1 + category_affinity × 5.0)
  │
  ├── [8] Фасетная агрегация (категории в результатах)
  │
  └── Ранжированные результаты с подсветкой и пояснениями
```

## Персонализация

Система строит **профиль каждой организации** на основе истории контрактов (2M+ записей):

- Для каждого заказчика (по ИНН) рассчитывается **category_affinity** — доля каждой категории в закупках
- При поиске, товары из «любимых» категорий получают **мультипликативный буст** до ×6.0
- Демо: запрос «шприц» от **Аптечного склада ДЗМ** → иммунодепрессанты в шприцах первыми (+146%)
- Тот же «шприц» от **Автохозяйства** → обычные медицинские шприцы

## Метрики качества

| Метрика | Значение |
|---|---|
| Hit@3 | **0.807** |
| MRR@10 | **0.807** |
| NDCG@10 | **0.801** |
| Precision@3 | 0.807 |
| Recall@10 | 0.904 |

Evaluation на **83 тест-кейсах** (8 ручных + 75 автогенерированных из реальных контрактов).
Graded NDCG: 2 = точное совпадение товара, 1 = совпадение категории, 0 = промах.

### Скорость

| Метрика | Значение |
|---|---|
| Среднее время поиска | **162мс** (35 разнообразных запросов) |
| Быстрые запросы | < 30мс (шприц, маска, мыло) |
| Повторные запросы (кэш) | < 0.1мс |
| Сложные запросы | < 400мс (кабель USB Type-C, стройматериалы) |

## Технологический стек

| Компонент | Технология |
|---|---|
| Backend API | FastAPI + Uvicorn |
| Web UI | Streamlit |
| Поисковый движок | SQLite FTS5 (BM25) |
| Семантический поиск | sentence-transformers (paraphrase-multilingual-MiniLM-L12-v2) |
| Морфология | pymorphy3 (LRU cache 4096) |
| Нечёткий поиск | rapidfuzz + pymorphy3 |
| Синонимы | 158 групп (data/synonyms.json) |
| Кэширование | 3 уровня: токены → LRU(512) → результаты(256) |
| Персонализация | Контрактная история → category affinity → multiplicative boost |
| Хранилище | SQLite (WAL mode, 64MB cache) |
| Python | 3.12+ |

> Архитектура намеренно **легковесная**: без GPU, без Elasticsearch, без внешних API. Всё работает на одной машине с SSD за < 200мс.

## Структура проекта

```
main.py                  # Точка входа (uvicorn)
streamlit_app.py         # Web UI (Streamlit)
requirements.txt         # Зависимости
app/
  api.py                 # FastAPI endpoints (22+ маршрутов: search, catalog, metrics)
  schemas.py             # Pydantic модели (Product, SearchResponse, CategoryFacet, ...)
  search.py              # Поисковый пайплайн: BM25 + семантика + персонализация + кэш
  repository.py          # SQLite-backed репозиторий (FTS5, vocabulary, facets)
  catalog_loader.py      # Загрузка CSV → SQLite (542K СТЕ, 2M контрактов)
  text_processing.py     # Токенизация, морфология, коррекция опечаток
  synonyms.py            # Расширение синонимами (158 групп)
  semantic.py            # Семантический поиск (sentence-transformers, 6506 category embeddings)
  reranker.py            # Heuristic reranker
  evaluation.py          # 83 тест-кейса (8 ручных + 75 авто), 5 метрик, graded NDCG
  evaluation_compare.py  # Сравнение режимов поиска
  demo_scenarios.py      # 5 demo-сценариев для жюри
  settings.py            # Конфигурация
data/
  СТЕ_*.csv             # 542K товаров (ID, title, category, attrs)
  Контракты_*.csv       # 2M контрактов (11 полей)
  synonyms.json          # 158 групп синонимов
  category_embeddings.pkl # Кэш эмбеддингов категорий
```

## Быстрый старт

```bash
# Клонирование
git clone https://github.com/gpudreamthaw/smart-search-hackathon.git
cd smart-search-hackathon

# Виртуальное окружение
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Linux/Mac

# Установка зависимостей
pip install -r requirements.txt

# Запуск API-сервера (порт 8000)
uvicorn main:app --reload

# Запуск Web UI (порт 8501, в отдельном терминале)
streamlit run streamlit_app.py
```

При первом запуске система загружает CSV-файлы из `data/` в SQLite (`smart_search.db`).
